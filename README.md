# 🤖 VPN Telegram Bot

Telegram-бот для автоматической выдачи VPN-ключей (WireGuard / Shadowsocks / Xray).  
Пользователь отправляет команду — бот генерирует конфиг, отправляет файл и/или QR-код.

## ✨ Возможности

- Генерация VPN-конфигов по запросу
- Поддержка нескольких протоколов (WireGuard, Shadowsocks, VMess/VLESS)
- Отправка `.conf`-файла и QR-кода для мобильных устройств
- Автоматическое добавление пользователя в базу данных
- Просмотр статистики подключений (опционально)
- Поддержка промокодов или реферальной системы (опционально)
- Работает на Python + aiogram (или pyTelegramBotAPI)

## 📦 Требования

- Сервер (VPS) с Ubuntu 20.04 / 22.04
- Установленный WireGuard / Shadowsocks / Xray (в зависимости от выбранного протокола)
- Python 3.9+
- Библиотеки: `aiogram`, `requests`, `qrcode`, `python-dotenv` и др.
- Токен бота от [@BotFather](https://t.me/BotFather)

## 🚀 Установка

### 1. Клонируйте репозиторий
```bash
git clone https://github.com/ваш-username/vpn-telegram-bot.git
cd vpn-telegram-bot
```

### 2. Установите зависимости
```bash
pip install -r requirements.txt
```

### 3. Настройте переменные окружения
Создайте файл `.env` в корне проекта:
```env
BOT_TOKEN=ваш_токен_бота
VPN_TYPE=wireguard   # или shadowsocks, xray
SERVER_PUBLIC_KEY=...
SERVER_PRIVATE_KEY=...
SERVER_ENDPOINT=ваш_ip:порт
```
> Все параметры подробно описаны в `.env.example`.

### 4. Запустите бота
```bash
python bot.py
```

Для запуска в фоновом режиме используйте `systemd` или `screen`.

## 🖥️ Использование

| Команда       | Описание |
|---------------|----------|
| `/start`      | Приветственное сообщение |
| `/help`       | Справка по командам |
| `/vpn`        | Получить новый VPN-конфиг |
| `/status`     | Проверить статус подключения (если реализовано) |
| `/renew`      | Обновить ключ (если разрешено) |

После отправки `/vpn` бот сгенерирует конфигурацию и пришлёт её в виде файла, а также QR-код (для WireGuard).

## ⚙️ Конфигурация

Все настройки хранятся в `.env`. Пример:

```env
BOT_TOKEN=123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11
VPN_TYPE=wireguard
WG_CONFIG_PATH=/etc/wireguard/wg0.conf
WG_CLIENT_PATH=/etc/wireguard/clients
SERVER_ENDPOINT=192.168.1.1:51820
SERVER_PUBLIC_KEY=your_server_public_key
```

Если вы используете Shadowsocks, укажите:
```env
VPN_TYPE=shadowsocks
SS_SERVER_ADDR=ваш_ip
SS_SERVER_PORT=8388
SS_PASSWORD=пароль
SS_METHOD=chacha20-ietf-poly1305
```

## 📝 Примечания

- Бот не хранит логи пользователей, все данные сохраняются только на сервере.
- Для безопасности рекомендуется использовать `systemd`-сервис и настроить автоматический перезапуск.
- При возникновении ошибок проверьте права доступа к конфигурационным файлам WireGuard.
